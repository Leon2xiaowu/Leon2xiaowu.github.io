Webpack学习笔记一

· 2017-02-13 · # Webpack # Javascript

按照webpack-demos的顺序进行了学习，结合自己的理解简单的翻译了一下，食用原repository更佳,点我看原repo,给作者个star

What is Webpack

Webpack是一个前端打包工具，能够将模块和依赖进行整合输出。

Webpack的配置文件是webpack.config.js

module.exports = {
  entry: "entryFileUrl",
  output: {
    filename: "outputFileUrl"
  }
  //.....
}

当你游了配置文件后，就可以使用webpack命令进行编译，一些基本的命令行指令是你需要知道

  • webpack 在开发环境下执行一次编译
  • webpack -p 在生产环境下执行一次编译，对源码进行压缩混淆
  • webpack --watch 对程序进行监听并持续打包编译
  • webpack -d 打包时生成map映射文件
  • webpack --colors 更美观？

你也可以在package.json配置文件中的“scripts”进行配置

  "scripts": {
    "dev": "webpack-dev-server",
    "deploy": "NODE_ENV=production webpack -p"
  },

Entry file and build

entry告诉webpack根据哪一个文件进行打包，output则表示打包编译后输出文件的目录，for example

 module.exports = {
   entry: "./main.js",
   output: {
     filename: "build.js"
   }
 }

main.js就是entry file，build.js就是经过打包后输出的文件。

多入口文件

Webpack允许多入口文件的配置，这可以用于多页面应用中。 example

  module.exports = {
    entry: {
      "build1": './main1.js',
      "build2": './main2.js'
    },
    output: {
      filename: '[name].js'
    }
  }

entry由字符串变成对象，并在其中添加[编译名的文件名]:[入口文件Url]组成的键值对； output由指定的文件名变成[name].js，根据entry指定的name来打包编译。

Loaders

Loaders也叫加载器，它能够为你的应用对源文件做进一步的处理。

Babel-loader

打个比方，Babel-loader能够将JSX/Es6转换成浏览器能够识别的js文件。

  module.exports = {
    entry: "./main.js",
    output: {
      filename: "build.js"
    },
    module: {
      Loaders: [
        // ===Babel加载器===begin
        {
          test: /\.js$/,
          exclude: /(node_modules|bower_components)/,
          loader: 'babel-loader',
          query: {
            presets: ['es2015']
          }
        }
        //===end
      ]
    }
  }

Css-loader

在js中有时需要加载css样式表文件，Css-loader可以胜任这项工作。

module.exports = {
  entry: './main.js',
  output: {
    filename: 'bundle.js'
  },
  module: {
    loaders:[
      { test: /\.css$/, loader: 'style-loader!css-loader' },
    ]
  }
};

Image-loader

在js中引用img文件

//..省略其他部分
loaders:[
  { test: /\.(png|jpg)$/, loader: 'url-loader?limit=8192' }
]

url-loader会将图片进行转换，如果图片小于8192bytes，将会转换成Base64的Data URL；反过来，就转换成正常的路径。 假设有两张一大一小的图片，第一张小于8192bytes，第二张则大于。那么两者会被换成这样：

<!-- 小图转换成Base64 -->
<img src="data:image/png;base64,iVBOR...uQmCC">  
<img src="4853ca667a2b8b8844eb2693ac1b2578.png">

Css Module

css-loader?modules 能够开启加载的Css模块中的属性的作用域。 也就是说你导入的css模块中的属性默认是本地的、局部的，你能选择使用:global(...)将选择器提升为全局。详细 index.heml

<html>
<body>
  <h1 class="h1">Hello World</h1>
  <h2 class="h2">Hello Webpack</h2>
  <div id="example"></div>
  <script src="./bundle.js"></script>
</body>
</html>

app.css

.h1 {
  color:red;
}

:global(.h2) {
  color: blue;
}

main.JSX

var React = require('react');
var ReactDOM = require('react-dom');
var style = require('./app.css');

ReactDOM.render(
  <div>
    <h1 className={style.h1}>Hello World</h1>
    <h2 className="h2">Hello Webpack</h2>
  </div>,
  document.getElementById('example')
);

webpack.config.js

module.exports = {
  entry: './main.jsx',
  output: {
    filename: 'bundle.js'
  },
  module: {
    loaders:[
      {
        test: /\.js[x]?$/,
        exclude: /node_modules/,
        loader: 'babel-loader',
        query: {
          presets: ['es2015', 'react']
        }
      },
      {
        test: /\.css$/,
        loader: 'style-loader!css-loader?modules'  //注意⚠️ ?modules
      }
    ]
  }
};

第一个h1不会有任何变化，因为没有定义全局的.h1选择器； 第二个h1为红色，由于.h1的作用范围为本地，所以style.h1能够正确找到选择器； h2均为蓝色，其是通过:global(.h2){...}定义的选择器，所以作用范围为全部。 {%asset_img css-module.png 运行结果%}

Plugin

UglifyJs Plugin

Webpack 有它自己的插件用来扩展方法。比如UglifyJs plugin会最小化的输出编译过后的js

var webpack = require('webpack');
var uglifyJsPlugin = webpack.optimize.UglifyJsPlugin;
module.exports = {
  entry: './main.js',
  output: {
    filename: 'bundle.js'
  },
  plugins: [
    new uglifyJsPlugin({
      compress: {
        warnings: false
      }
    })
  ]
};

打包编译后，bundle.js会被最小化输出。类似于

var o="Hello";o+=" World",document.write("<h1>"+o+"</h1>")

HTML Webpack Plugin and Open Browser Webpack Plugin

webpack还可以使用第三方的插件。 HTML Webpack Plugin 能够为你自动生成一个index.html， Open Browser Webpack Plugin能够在webpack服务运行的时候打开一个新的浏览器标签并载入页面

var HtmlwebpackPlugin = require('html-webpack-plugin');
var OpenBrowserPlugin = require('open-browser-webpack-plugin');

module.exports = {
  entry: './main.js',
  output: {
    filename: 'bundle.js'
  },
  plugins: [
    new HtmlwebpackPlugin({
      title: "webpack-demo",
      filename: "index.html"
    }),
    new OpenBrowserPlugin({
      url: "http://localhost:8080"
    })
  ]
};

运行webpack-dev-save，现在你不需要自己创建index.html和打开浏览器了。👍

Environment flags

你还能利用环境标志，让部分代码只能运行在开发环境中 main.js

document.write('<h1>Environment flags</h1>')
if (__DEV__){
  document.write(new Date());
}

webpack.config.js

var webpack = require('webpack');

var devFlagPlugin = new webpack.DefinePlugin({
  __DEV__: JSON.stringify(JSON.parse(process.env.DEBUG || 'false'))
});

module.exports = {
  entry: './main.js',
  output: {
    filename: 'bundle.js'
  },
  plugins: [devFlagPlugin]
};

现在运行webpack

# Linux $ Mac
$ env DEBUG=true webpack-dev-server

# Windows
$ set DEBUG=true
$ webpack-dev-server

Code splitting

在大型项目中，将所有的代码放在一个文件中是极其没有效率的一件事件。webpack允许将其分割成许多不同的块，并按照需求进行加载。 首先，用require.ensure定义一个分割点

require.ensure(['./a'],function(require){
  vat content  = require('./a')
  document.open()
  document.write('<h1>Environment flags</h1>')
  document.colors()
})

require.ensure 告诉webpack "a.js"应该被分开加载并打包为一个文件。 另外一种方法是使用bundle-loader

// main.js

// Now a.js is requested, it will be bundled into another file
var load = require('bundle-loader!./a.js');

// To wait until a.js is available (and get the exports)
//  you need to async wait for it.
load(function(file) {
  document.open();
  document.write('<h1>' + file + '</h1>');
  document.close();
});
